7. LOCAL IMAGE STORAGE - EXACT FLOW
===================================

When admin uploads a service image:

1. Frontend admin page sends multipart upload
File:
D:\LinkedINAIAgent\ai-lead-agent\frontend\components\ServiceAdminPanel.tsx

2. Next admin API proxy forwards it
File:
D:\LinkedINAIAgent\ai-lead-agent\frontend\app\api\admin\services\[serviceId]\image\route.ts

3. Backend saves the file locally
File:
D:\LinkedINAIAgent\ai-lead-agent\backend\routes\services.py

4. Only image_path is stored in DB
File:
D:\LinkedINAIAgent\ai-lead-agent\backend\models\service.py

5. Public services page fetches and renders the local image
Files:
- D:\LinkedINAIAgent\ai-lead-agent\frontend\components\ServiceCatalog.tsx
- D:\LinkedINAIAgent\ai-lead-agent\frontend\lib\api.ts

12. SAFE CUSTOMIZATION ORDER
============================

If you want to customize without breaking things, use this order:

1. Change text and labels first
2. Change colors/layout second
3. Change env values third
4. Change backend model/schema only after that
5. Restart backend and frontend after env or model changes
